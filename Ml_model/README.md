# AI-Based Medical Report Analysis and Diagnosis Support System

This folder contains a standalone ML module for training and serving a medical report disease predictor.

## Features

- PDF text extraction using `pdfplumber` with PyMuPDF fallback
- TXT and image report extraction, with OCR for PNG/JPG files when Tesseract is installed
- Text preprocessing and tokenization
- Entity extraction for symptoms, diseases, medications, and lab values
- Baseline disease classification using `scikit-learn`
- Optional image disease classification from labeled image folders or CSV metadata
- Standalone Flask API for testing and later integration

## Structure

```text
Ml_model/
  dataset/
  models/
  nlp_model/
  training/
  utils/
  app.py
  predict.py
  train.py
  requirements.txt
```

## Install

```bash
pip install -r Ml_model/requirements.txt
```

Optional transformer-based NER:

```bash
set ENABLE_MEDICAL_NER=1
```

## Train

```bash
python -m Ml_model.train
```

Optional dataset path:

```bash
python -m Ml_model.train --dataset path/to/reports.csv
```

Train only one task:

```bash
python -m Ml_model.train --task report
python -m Ml_model.train --task symptom
python -m Ml_model.train --task image --dataset path/to/image_dataset
```

Training writes model artifacts under `Ml_model/models/` and updates `Ml_model/models/metrics.json` with:

- task name
- selected model
- dataset size
- labels
- cross-validation strategy
- metrics leaderboard
- model artifact path
- generation timestamp

Image datasets can be organized either as folders:

```text
image_dataset/
  Pneumonia/
    case_001.jpg
  Tuberculosis/
    case_002.png
```

or as a CSV with `image_path,label` columns. Relative image paths are resolved from the CSV file location.

Validate the default image dataset before training:

```bash
python -m Ml_model.validate_image_dataset
```

## Run API

```bash
python -m Ml_model.app
```

API endpoint:

```http
POST /analyze-report
```

Form field:

- `file`: PDF, TXT, PNG, JPG, or JPEG medical report

Response shape:

```json
{
  "prediction": "Pneumonia",
  "confidence": 0.84,
  "extracted_symptoms": ["cough", "fever", "shortness of breath"],
  "explanation": "The model suggests Pneumonia with 84.0% confidence based on symptoms such as cough, fever, shortness of breath and report findings including oxygen_saturation: 91.",
  "entities": {
    "symptoms": ["cough", "fever", "shortness of breath"],
    "diseases": ["pneumonia"],
    "medications": [],
    "lab_values": ["oxygen_saturation: 91"]
  },
  "probabilities": {
    "Pneumonia": 0.84
  }
}
```

## Metrics

Training reports:

- Accuracy
- Precision
- Recall
- F1 Score

Use `metrics.json` as the lightweight model lifecycle record for reviews, retraining notes, and CI smoke validation. The file is generated by training; it is safe to regenerate when datasets or candidate models change.

## Fallback Behavior

If a trained text artifact is unavailable, prediction falls back to keyword/heuristic scoring while preserving the same response shape. Image-only prediction returns an explanatory fallback until an image model artifact is trained.

## Smoke Tests

From the project root:

```bash
python -m unittest discover -s Ml_model/tests
```

The smoke tests verify report, symptom, heuristic fallback, and metrics metadata shapes.

## Notes

- The included dataset is a small starter dataset for demo and development.
- The text starter datasets include report/symptom examples for diabetes, hypertension, pneumonia, anemia, heart disease, dengue, asthma, stroke, chikungunya, diarrheal disease, COVID-19, tuberculosis, malaria, typhoid, UTI, kidney disease, liver disease, and thyroid disorder.
- The image model is optional because no labeled medical image dataset is committed. Add a local dataset and run `python -m Ml_model.train --task image --dataset path/to/image_dataset`.
- The module is isolated from the main app until you choose to integrate it.
